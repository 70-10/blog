# ブログ記事の文体・構成パターン分析レポート

分析対象: src/content/posts/ 内の全 98 記事

## 1. 文末表現の傾向

### **断定形（「〜だ。」「〜である。」）が主流**
- 全体の約 70-80%が断定形を使用
- 例：「原因は新しさを追い求めていることだろう。」「これは関西人の悲しい性でしょうね。」
- 技術記事・日記記事の両方で一貫して使用

### **丁寧語はほぼ使用しない**
- 「〜です。」「〜ます。」調は使用頻度が極めて低い
- 一部のチュートリアル記事（手順説明）でのみ「〜します。」を使用
  - 例：「npm install lit」「vitest.config.ts を設定します。」

### **特徴的な文末パターン**
- **「〜だろう」「〜だろうか」** - 推測・自問形式を多用
  - 例：「技術も毎日新しいことを覚えないと焦りを感じながら、これまでずっと走り続けてきた。」
- **「〜と思う」「〜と感じる」** - 主観的表現
  - 例：「内容が色あせないってすごい。」「来年もマイペースに頑張りたいです。」
- **体言止め** - 簡潔さを重視
  - 例：「問題となるのは、コミュニケーションのほうだ。」
- **倒置法** - 強調したいポイントで使用
  - 例：「読者は答えをすぐにでも知りたがっている。」

## 2. 段落構成の特徴

### **短い段落を好む**
- 1 段落は平均 2-4 文程度
- 1 文 1 段落も頻繁に使用（強調したいポイント）
- 例：
  ```
  けど、経験を積むにつれて新しさの感覚は徐々に薄れていく。

  10年目を超えるころにはひと通りの経験はしてしまっている。
  ```

### **見出しの使い方**
- **H1（#）**: 大きなセクション区切り
  - 「はじめに」「感想」「参考情報」など
- **H2（##）**: 具体的な論点・項目
  - 結論を見出しに書く傾向が強い
  - 例：「エモくてセンチメンタル」「実践したくなる」「普遍的な関数は自分で作らなくてもいい」
- **H3（###）以降**: 詳細な分類・補足
  - チュートリアル記事で手順説明に使用

### **箇条書きの使用**
- リスト表示を積極的に活用
- 複数の項目を列挙する際に必ず使用
- 番号付きリスト（1, 2, 3...）と記号リスト（-）を使い分け
  - 手順・順序がある場合：番号付き
  - 並列関係の項目：記号

## 3. 語り口・表現方法

### **個人的な経験・感想の記述スタイル**
- **一人称「自分」を多用**
  - 例：「自分は車輪の再発明に陥りやすいので」「当時の自分はこの言葉にピンとこなかった」
- **率直な感情表現**
  - 「なるほど」「すごい」「おもしろい」「やるせない」「切ない」
- **内省的・自己分析的**
  - 経験を振り返り、自分の変化を言語化する傾向が強い

### **具体例の提示方法**
- **コード例を必ず含める**（技術記事）
  - before/after 形式でリファクタリング結果を示す
- **実際の体験談を語る**（日記記事）
  - 時系列で展開し、「当時→今」の対比を行う
- **他者の言葉を引用**
  - 先輩の言葉、書籍の一節、スライドの内容など

### **引用の使い方**
- **書籍からの引用**
  - blockquote で囲み、出典を明示
  - 例：「> 行動を変え、新たな一歩を踏み出すのに「遅すぎる」ということはない。」
- **URL リンクの記載方法**
  - 本文に埋め込む形式と、独立した行で表示する形式を併用
  - 参考情報セクションで一覧化することが多い

## 4. 記事の構成パターン

### **技術記事の典型的な構成**
1. **導入部**: 何を作った/試したか（1-2 段落）
2. **実装手順**: コード例を交えて説明（見出しで分割）
3. **結果・感想**: 使ってみた所感
4. **参考情報**: リンク集

**例：「Vitest で Lit コンポーネントをテストするためのチュートリアル」**
- プロジェクトを用意する → Lit をインストールする → Vitest をインストール... という流れ

### **書評・読書記事の構成**
1. **はじめに**: 読んだ本の紹介
2. **感想**: 複数の小見出しで論点を整理
   - 「驚いたこと」「共感したこと」「わかっていないこと」など
3. **さいごに**: 自分の行動変容や決意

### **考察・エッセイ記事の構成**
1. **問題提起**: 現在の状況や疑問を提示
2. **展開**: 過去の経験や他者の意見を織り交ぜながら考察
3. **結論**: 自分なりの答えや今後の方向性

**特徴**: 「〜だろうか」という自問から始まることが多い

### **まとめ記事・リスト記事**
- 「X の実装パターンまとめ」「よく使う X のリスト」形式
- 見出しで項目を分割し、それぞれに説明を加える
- 最後に「どれを使うべきか」という選択ガイドを提供

## 5. 特徴的な表現や言い回し

### **よく使われる接続詞・副詞**
- 「けど」「ただ」「なので」- 逆接・順接を多用
- 「なるほど」「確かに」- 納得・肯定の表現
- 「というのも」「つまり」- 説明・言い換え
- 「ただ」で文を始める（逆接の強調）

### **特徴的なフレーズ**
- 「〜してみた」「〜してみる」- 試行・実験のニュアンス
- 「〜だとわかりました」「〜ということがわかった」- 学びの明示
- 「〜といった」「〜のような」- 例示
- 「〜すぎる」- 程度の強調（「読みすぎ問題」など）
- 「〜ばかり」- 限定・頻度の強調

### **比喩・メタファー**
- 「車輪の再発明」「芽を摘む」など定番の比喩を使用
- 技術的な概念を日常的な言葉で説明する工夫

## 代表的な記事の例

### 1. **「10 年前の君から」** - エッセイ記事
- **特徴**: 内省的、時間軸での変化を描写
- **文体**: 「〜だ」調、短い段落
- **構成**: 過去の言葉の引用 → 当時の感想 → 現在の状況 → 気づき

### 2. **「else-if を書かなくなった理由」** - 考察記事
- **特徴**: 問いから始まり、根拠を探し、結論を導く
- **文体**: 論理的、引用を効果的に使用
- **構成**: 問題提起 → 過去の学習経験 → 書籍からの引用 → 自己分析 → 結論

### 3. **「Vitest で Lit コンポーネントをテストするためのチュートリアル」** - 技術チュートリアル
- **特徴**: 手順を明確に、コード例を豊富に
- **文体**: 「〜します」調（手順部分）と「〜だ」調（説明部分）の混在
- **構成**: 手順を番号付きリストで明示、各ステップでコード例

### 4. **「自分のことばで」** - 短文エッセイ
- **特徴**: 短く、メッセージ性が強い
- **文体**: 「〜だ」調、短い段落の連続
- **構成**: 問題提起 → 対比 → 決意

### 5. **「Astro 5 にアップデートした」** - 実装報告
- **特徴**: まず全体像を箇条書きで提示、その後詳細説明
- **文体**: 「〜だ」調、「〜しました」（報告）
- **構成**: 概要 → アップデート内容のまとめ（箇条書き）→ 各項目の詳細

## 文体再現のためのガイドライン

### 基本ルール
1. **文末は「〜だ」「〜である」調を基本とする**
   - 例外：手順説明では「〜します」も可
2. **短い段落を心がける（2-4 文）**
   - 強調したい一文は単独段落にする
3. **見出しには結論を書く**
   - 「X とは何か」ではなく「X は Y だ」
4. **一人称は「自分」を使用**
   - 「私」「僕」は避ける（日記記事では「僕」も使用可）

### 語彙・表現
5. **「けど」「ただ」「なので」を多用**
   - 自然な口語調のリズムを作る
6. **「〜だろう」「〜だろうか」で推測・自問**
   - 断定しすぎない柔らかさを出す
7. **率直な感情表現を使う**
   - 「すごい」「おもしろい」「なるほど」
8. **「〜してみた」形式で試行性を表現**
   - 実験的・探索的な姿勢を示す

### 構成
9. **技術記事は「導入 → 手順 → 感想 → 参考情報」**
10. **考察記事は「問い → 経験/引用 → 考察 → 結論」**
11. **箇条書きを積極的に使う**
    - 並列項目は記号リスト、順序があるなら番号リスト
12. **コード例は惜しまず載せる**
    - before/after、完成形を必ず提示

### 避けるべき表現
13. **過度な丁寧語は使わない**
    - 「〜でございます」「恐れ入りますが」など
14. **装飾的な表現を避ける**
    - シンプルで直接的な言葉を選ぶ
15. **冗長な説明をしない**
    - 「先ほど申し上げたように」「前述の通り」など

### 読者への配慮
16. **「読者は答えを知りたい」を念頭に**
    - 結論を先に、詳細は後に
17. **自分の学びを明示する**
    - 「〜ということがわかった」「〜と感じた」
18. **未来の自分へのメモとして書く**
    - 他者に教えるより、自分の記録として

---

このガイドラインに従うことで、ブログ記事の文体を再現できます。率直で、内省的で、技術的でありながら人間的な温度を持つ文章スタイルです。
